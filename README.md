# astrbot_plugin_wecomocr

这是一个用途受限的 AstrBot 插件：接收一张离校清单图片或 PDF，通过百度 AI Studio 的 `PaddleOCR-VL-1.6` 提取信息，让用户确认或修改，最后调用 WPS AirScript 填表。

## 工作流程

1. 白名单平台实例或精确来源中的用户上传一张 JPG、JPEG、PNG 或 PDF。
2. 如果同一条消息包含多个文件，插件只处理第一个，并明确告知其余文件已忽略。
3. 插件返回七个确认字段：姓名、工号、单位、复旦邮箱、离职日期、保留邮箱、邮箱注销日期。
4. 若 OCR 得到合法离职日期且复旦邮箱不为 `无`，插件默认将邮箱注销日期设为离职日期后第 31 天；若未得到合法日期，则在用户第一次把离职日期修改为合法日期时计算一次。
5. 用户可回复 `姓名改为张三`；多个修改可用换行或分号分隔。保留邮箱只允许填写 `是` 或 `否`。
6. 用户回复 `提交`、`确认提交` 或 `确认无误` 后，插件映射 WPS 字段并清空本轮状态。

白名单平台实例下的所有会话（包括之后新建的客服会话）以及精确来源白名单中的消息，都会被插件接管并阻止默认自由对话 LLM。确认阶段会先使用本地规则解析修改；启用 `enable_llm_intent_parsing` 后，只有本地规则无法理解时才以无工具、无历史上下文方式调用当前会话模型，并对返回字段再次执行白名单和格式校验。识别前的无关请求会直接拒绝；识别后的错误输入只会报错并保留本轮状态。用户可回复 `退出`、`取消本轮` 或 `重新开始` 主动清空本轮。

## 安装与配置

安装插件后，在 AstrBot 插件配置页填写：

- `allowed_platform_ids`：需要完整接管的平台适配器实例 ID（推荐）。配置后，该实例传入的所有 UMO 都会进入本插件，即使把微信客服发给新用户而产生了新 UMO，也无需再次配置。AstrBot 的平台实例 ID 是唯一的，因此不会误接管另一个同类型适配器实例。
- `allowed_sources`：可选的精确 `unified_msg_origin` 列表，通常格式为 `平台ID:消息类型:会话ID`。仅在只想放行特定聊天时使用；它与 `allowed_platform_ids` 取并集。两个白名单都为空时拒绝全部来源。
- `allowed_sender_ids`：可选的发送者 ID 白名单。留空表示白名单来源中的所有发送者均可使用。
- `baidu_api_key`：百度 AI Studio PaddleOCR API Key。
- `wps_script_url`：完整的 WPS AirScript 同步任务地址，例如 `https://365.kdocs.cn/api/v3/ide/file/.../script/.../sync_task`。
- `airscript_token`：WPS 的 `AirScript-Token`。
- `enable_llm_intent_parsing`：确定性解析失败后是否调用当前会话模型理解自然语言修改，默认开启。
- `llm_intent_timeout`：自然语言修改识别超时，默认 30 秒。
- `cache_retention_hours`：插件处理缓存保存时长，默认且最少 1 小时，可延长。
- `cache_cleanup_interval_minutes`：异常重启后过期缓存的兜底扫描间隔。

其余超时、模型及调试选项通常保持默认即可。若暂时不知道平台实例 ID，可先开启 `debug`，发送一条消息后从 AstrBot 日志的 `platform_id=...` 中复制该值到 `allowed_platform_ids`，再关闭调试。UMO 第一个冒号前的部分通常也是这个 ID。

> 配置页中的 API Key 和 Token 属于敏感信息。不要提交到 Git，也不要发送给普通用户；应限制 AstrBot 配置文件及日志的访问权限。

## WPS 脚本入参

插件仅向 `https://365.kdocs.cn` 域名且以 `/sync_task` 结尾的接口发送请求。通用字段中，内部的 `单位` 会映射成 WPS 的 `原院系单位`。

当 `保留邮箱=否` 时提交：

```json
{
  "Context": {
    "argv": {
      "data": {
        "姓名": "张三",
        "工号": "20260001",
        "原院系单位": "某单位",
        "复旦邮箱": "zhangsan@fudan.edu.cn",
        "离职日期": "2026.07.16",
        "邮箱注销日期": "2026.08.16"
      }
    }
  }
}
```

当 `保留邮箱=是` 时不提交邮箱注销日期，改为额外提交：

```json
"保留姓名": "张三"
```

## 行为与限制

- 会话按“消息来源 + 发送者 ID”隔离，内存中只保存当前待确认的七个字段和注销日期初始化状态。
- 用户直接修改邮箱注销日期后，后续修改离职日期不会覆盖该值。
- 保留邮箱为 `否` 时，提交前必须存在真实有效的邮箱注销日期。
- 插件重载或 AstrBot 重启会清空未提交会话。
- 确认阶段再次上传文件、发送无关文本或输入无效字段值时只会报错，不会清空已识别信息。
- 连续错误达到 `review_error_hint_threshold`（默认 3）次后，错误回复会附加修改、提交和退出提示；成功修改后重新计数。
- 用户可回复 `退出`、`取消`、`取消本轮`、`结束本轮`、`清空本轮`、`重新开始` 或 `重置` 主动结束本轮。
- 修改支持中文逗号、英文逗号或换行分隔，也支持“把姓名换成李四”“单位应该是发展研究院”等自然表达。`邮箱`、`邮件` 和 `email` 均作为 `复旦邮箱` 的别名。
- 复旦邮箱修改为 `无`、`无邮箱`、`没有邮箱` 或 `留空` 时，邮箱注销日期会自动清空，WPS 中对应字段提交空字符串。
- 开启 LLM 意图识别会把当前七字段和本条用户消息发送给 AstrBot 当前会话模型；请按所在单位的数据合规要求决定是否开启。
- 上传内容会复制到插件专属临时缓存，默认 1 小时后删除；插件重启后会扫描并删除已过期缓存。插件不会主动删除 AstrBot 或消息平台持有的原始文件。
- WPS 请求超时可能代表结果未知。插件会清空本轮并提示管理员先检查表格，以避免重复提交。
- OCR 依赖 `requests`，由 `requirements.txt` 声明；不依赖本地 PaddlePaddle/PaddleOCR 环境。
